Research on Recommender Systems

  • Machine Learning

  • Pandas

Predicted vs Actual Ratings: User ratings predicted by each of the algorithms (left to right: Random, Baseline, SVD, KNN) are plotted against the actual ratings by users in the ml-small data set. I used matplotlib to produce the chart.

In summer 2022 I did machine learning research at Cornell with Professor Sarah Dean. I created a graph with three node layers that represent the multiple stakeholders in a Recommender system: the providers that supply the items, the items which are recommended, and users who receive the item recommendations. I defined item and provider visibility, which is a graph-based measure that represents the number of users that sees an item or a provider through the recommendations that they receive from the system. By measuring and comparing the visibilities of items and providers that are recommended by various recommendation algorithms, I found that common collaborative filtering methods show popularity bias for providers similarly to items.

Avatar Game

  • OCaml
  • Functional Programming
  • Agile Methodology

Battle End: The battle_end function updates the player's states and allows the player to choose a new move that they can replace an old move with. I used pattern matching to efficiently update the desired moves and strings, according to user input. As the game takes in user input, it is important to handle errors and exceptions that may arise from incorrect or unexpected user input.

As part of the course CS 3110: Functional Programming and Data Structure, I worked in a team of four students to create a text-based game based on the Avatar: The Last Airbender series. We designed the game to have single and multi player modes, as well as a continuous storyline following the original series. In the single player mode, the player encounters a new opponent at each new "episode" that they must beat using a set of moves in order to move on to the next episode. In the multi player mode, up to 4 players participate in a 2 vs 2 battle against other players or AI. As we implemented the game, we focused on modularity to reuse as much code as possible across all of the different modes of playing. We wrote all of the story dialogues and the character details in a standard JSON format to make the game easily expandable with more stories and characters. I specifically developed the build files, the testing interface, as well as the main gameplay of both the single and multi player modes.
